## Description

Momento is a private scrapbook for physical collectibles.

Scan supported objects into detailed USDZ 3D models with guided Object Capture, preview them in-app, and open them in AR. Add supporting photos, voice memos, text memories, tags, purchase details, estimated values, serial numbers, and provenance notes.

Your archive is local-first. Momento stores metadata with SwiftData and keeps large assets, including models, photos, and audio files, in app-managed local storage. Nothing is public by default.

Key features:

- Guided 3D object capture for supported iPhone and iPad devices
- On-device reconstruction to USDZ
- In-app 3D preview and AR Quick Look
- Shelf grid with search, filters, and sort options
- Photos, voice memos, notes, tags, and collection metadata
- Optional Face ID app lock
- Insurance-ready PDF and CSV exports
- Privacy-safe data archive exports with sanitized photo copies
- Local-first storage with no public sharing by default

Object Capture requires a supported device, good lighting, and steady movement around the object. Reconstruction can take time; keep Momento open while a model is processing.

## Review Notes

Momento is an iOS-only local-first collectible archiving app.

Core flows to review:

1. Complete onboarding.
2. Add a collection item.
3. On supported devices, use guided Object Capture to scan an object and reconstruct a USDZ model.
4. On simulator or unsupported hardware, use the app's non-capture metadata, attachment, shelf, settings, and export flows.
5. Open item detail to view metadata, attachments, 3D preview, and AR Quick Look when a model exists.
6. Open Settings to test export and optional Face ID lock.

Privacy notes:

- Momento does not create public profiles or public sharing feeds.
- Metadata is stored locally with SwiftData.
- USDZ, photo, and audio assets are stored as local files in app-managed storage.
- Imported and exported photos are sanitized to remove GPS metadata where possible.
- Optional cloud suggestions are off by default and require the user to enter an HTTPS endpoint and consent before any downsampled image is sent.

Hardware notes:

- Object Capture requires Apple-supported hardware and iOS availability.
- If Object Capture is unsupported in the review environment, Momento shows an unsupported-device state rather than crashing.

## Permission Usage Explanations

Camera:
Momento uses the camera for guided Object Capture to create 3D digital twins of physical collectibles.

Microphone:
Momento uses the microphone only when the user records voice memos for a collection item.

Photos:
Momento uses photo library access when the user chooses photos to attach to collection items.

Face ID:
Momento uses Face ID only when the user enables the optional app lock.

## Privacy Nutrition Label Draft

Data collection:

- Data collected: None, unless final product decisions add analytics, accounts, sync, or crash reporting.
- Tracking: No.
- Third-party advertising: No.

Data stored on device:

- Collection item metadata
- USDZ model files
- Thumbnails
- User-selected photos
- Voice memos
- Text notes
- Export files generated by user action

Optional user-configured cloud suggestion endpoint:

- Off by default.
- Sends one downsampled object image only after the user enables cloud suggestions and provides an HTTPS endpoint.
- Do not mark this as app tracking.
- Revisit privacy answers if Momento ever ships a company-operated cloud endpoint.
